Clase Teorica

  1. Que hace un JS Engine

    • Recibe código fuente
    • Parsea el codigo y produce un
      Abstract Syntax Tree (AST)
    • Se compila a bytecode y se ejecuta
    • Se optimiza a machine code y se
      reemplaza el código base

  2. Parser

    • descompone el código fuente en tokens
      al obtener los tokens pasa a Abstract Syntax Tree

  3. Google dice:

    • Parsin es del 15-20% del proceso de ejecución
    • La mayoria del JavaScript en una página nunca se ejecuta
    • Esto hace bundling y code splitting sea muy importante!

  4. Parser de V8

      Eager Parsing:
    • Encuentra errores de sintaxis
    • Crea el AST
    • Construye scopes
      • Lazy Parsing:
    • Doble de rápido que el eager parser
    • No crea el AST
    • Construye los scopes parcialmente

  5. Abstract Syntax Tree (AST)

    • Es un grafo (Estructura de datos)
      que representa un programa

  6. Bytecode vs Machine code

      Bytecode:
    • Codigo parecido a assembly
    • portatil
    • Ejecutado por una maquina virtual
      • Machine Code:
    • Binario
    • Instrucciones especificas a una arquitectura o un procesador
